##编码规范的生效日期有组长通知##S

#代码风格

    -花括号{}在任何情况下不允许省略
    -花括号独占一行，如
        -public methodName（）
        {
        //代码段
        }
    -使用括号"()"强调运算优先级
    -注意正确缩进
    -除非算法需要，不允许使用float

#命名规范

    -类名使用名词及名词性短语，小驼峰命名法（如书上）
    -方法名使用动词及动词短语，小驼峰命名法（如书上）
    -变量名，引用变量名使用匈牙利命名法
    -除非算法需要，否则方法体中必须使用this
    -常量名全部大写


#注释

    -对接口和复杂的代码块必须注释，并注明最近一次修改的信息（时间，原因等）
    -修改代码时保持注释同步
    -未完成的方法使用中文注释强调未完成
    -修改他人代码是要先注释对方代码，并写明修改原因，不允许随便删除他人代码。
    -使用位运算的地方必须注释
    -发布前整理注释
